Jira AUTH-891 — Login fails on Safari, redirect loop after sign-in. In Progress. Users get stuck in a loop when signing in on Safari; the session cookie or redirect flow is broken for that browser. Fix the session cookie and redirect flow so Safari users can log in successfully.
Slack #auth-safari-bug — Scope: only Safari (confirmed). No reports from Chrome or Firefox. When you fix it, add some logging around that path so we can debug if it happens again. Consider adding a metric so we can alert if Safari login failures spike.
GitHub auth-lib/session #142 (closed) — Safari redirect loop with SameSite cookies. Resolved issue on the auth library we use. Solution from maintainer: set SameSite=None; Secure on the session cookie when using cross-site redirects. Requires HTTPS. Fixed in library v3.2; we may need to upgrade or apply the same cookie attributes in our app.
